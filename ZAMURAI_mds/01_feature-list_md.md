# 2️⃣ 優先順位の定義基準（共通ルール）

|**優先度**|**判断基準**|
|---|---|
|🔴 P0|これがないとプロダクトが成立しない（MVP必須）|
|🟡 P1|体験を大きく向上させる（アルファ〜ベータ版で追加）|
|🟢 P2|運用効率・継続率を改善する（ポリッシュ段階）|
|⚪ P3|将来の拡張性・差別化（リリース後のアプデ等）|

# 3️⃣ フェーズ整理テンプレート

Plaintext

```
【Phase 0 - MVP】
🔴 P0のみ実装
（ブロックとTポーズのキャラが、声で点数を稼ぎ、敵から逃げるだけの最小構成）

【Phase 1 - 低クオリティ】
🟡 P1追加
（PSXシェーダーが入り、アニメーションやおすすめワードなどの「遊び」が追加される）

【Phase 2 - 高クオリティ】
🟢 P2追加
（UIの作り込み、自分視点のアニメーションなど手触りの向上）

【Phase 3 - 拡張】
⚪ P3追加
（スキン変更、新マップ、新コマンドなど）
```
